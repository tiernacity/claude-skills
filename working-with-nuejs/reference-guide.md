# Nue.js 2.x Reference Documentation

When you need detailed specs beyond workflow patterns, consult official Reference docs at https://nuejs.org/docs/.

**⚠️ CRITICAL**: Only use Nue 2.x documentation (2024+). If unsure, verify examples use `<!doctype dhtml>` syntax.

## When to Consult Reference

**Use Reference for:**
- Complete `site.yaml` configuration options
- Full CLI command flags
- Comprehensive HTML syntax (all `:if`/`:for`/`@click` variations)
- Advanced layout patterns
- Complete State API specification
- Full Server API context methods
- Nuemark syntax details

## Key Reference Sections

**Configuration:** https://nuejs.org/docs/reference/configuration.html
All `site.yaml` options, environment settings, build config.

**Command Line:** https://nuejs.org/docs/reference/command-line.html
All `nue` commands, flags, environment variables.

**HTML Syntax:** https://nuejs.org/docs/reference/html-syntax.html
Complete directive reference, event handlers, data binding expressions.

**Layout System:** https://nuejs.org/docs/reference/layout-system.html
Layout hierarchy, slots, conditional layouts.

**Template Data:** https://nuejs.org/docs/reference/template-data.html
Accessing front matter, global data, collection data.

**State API:** https://nuejs.org/docs/reference/state-api.html
Advanced state management, computed properties, watchers.

**Server API:** https://nuejs.org/docs/reference/server-api.html
Complete context object methods, middleware patterns.

## Pattern

1. Use skill guides for workflow and patterns
2. When you hit specific question ("syntax for...?"), consult Reference
3. Reference docs are authoritative - skills are practical workflows
